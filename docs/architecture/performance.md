# Performance engineering

## Experience

- asynchronous and parallel execution;
- TPL and PLINQ;
- scale-up optimization as preparation for scale-out;
- memory and CPU profiling;
- startup and reflection-cost reduction;
- caching of metadata and type discovery;
- test-suite initialization and fixture optimization;
- build resource and packaging optimization;
- AWS Lambda and SnapStart considerations.

## Principle

Performance work begins with measurement. However, architecture should avoid known structural costs such as repeated reflection scans, unnecessary allocations, serial I/O, and duplicated initialization.

## Tooling history

Visual Studio profiling tools, Redgate ANTS Performance Profiler, Redgate ANTS Memory Profiler, load testing, application telemetry, and targeted benchmarking.
