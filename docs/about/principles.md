# Engineering principles

1. **Prefer explicit intent over clever code.**
2. **Turn repeated infrastructure into reusable platform capabilities.**
3. **Make the correct path the easiest path.**
4. **Design APIs for discoverability, consistency, and migration.**
5. **Treat testing, diagnostics, and documentation as product features.**
6. **Automate architectural rules where possible.**
7. **Measure performance before optimizing—but design with cost awareness.**
8. **Use conventions to remove noise, not to hide behavior.**
9. **Keep domain logic independent from infrastructure mechanics.**
10. **Make extension points intentional rather than accidental.**
11. **Avoid abstractions that merely rename complexity.**
12. **Continuously remove boilerplate from product teams.**
